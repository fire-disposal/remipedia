# 智能床垫设备集成方案

## 概述

本文档描述了如何将智能床垫设备集成到现有的IoT健康数据平台中，实现**事件驱动**的智能数据过滤，只存储有价值的事件（上床/下床、评分体动、定期测量），避免每秒2次高频数据带来的存储噪声。

## 技术架构

### 1. 数据接收层

**TCP服务器**
- 端口：5858（可配置）
- 协议：TCP/IP + MessagePack
- 数据包格式：
  ```
  [魔数: 0xABCD][数据长度: 1字节][CRC8校验: 1字节][MessagePack数据: N字节]
  ```

**主要特性**
- 异步处理，支持高并发连接
- 自动数据包边界检测和重组
- CRC校验确保数据完整性
- 错误恢复机制，跳过损坏数据包

### 2. 数据解析层

**智能床垫适配器**
- 支持MessagePack二进制格式解析
- 内置数据验证和降噪处理
- 实时翻身检测算法
- 数据类型：`mattress_status` 和 `turn_over_event`

**数据字段映射**
```rust
pub struct MattressData {
    pub manufacturer: String,    // Ma: 制造商 (固定为"HT")
    pub model: String,           // Mo: 型号 ("02"/"03")
    pub version: i32,            // V: 协议版本
    pub serial_number: String,   // Sn: 设备序列号
    pub firmware_version: i32,   // fv: 固件版本
    pub status: String,          // St: 体征状态 (on/off/mov/call)
    pub heart_rate: i32,         // Hb: 心跳频率
    pub breath_rate: i32,        // Br: 呼吸频率
    pub wet_status: bool,        // Wt: 尿湿状态
    pub apnea_count: i32,        // Od: 呼吸暂停次数
    pub weight_value: i32,       // We: 辅助重量值 (0-20, -1表示未安装)
    pub position: [i32; 2],      // P: 身体位置坐标 [头部, 胸部]
}
```

### 3. 智能数据过滤层

**事件驱动架构**
- **原始数据过滤**：每秒2次数据 → 只提取有价值事件
- **状态机管理**：离床 → 在床 → 体动 → 呼叫 状态转换
- **智能事件检测**：上床、下床、评分体动、定期测量

**有价值事件类型**

1. **上床事件** (`bed_entry_event`)
   - 重量值 > 15 且状态变为 "on"
   - 置信度计算：基于重量、心率、状态综合评估
   - 存储：时间戳、置信度、重量值

2. **下床事件** (`bed_exit_event`) 
   - 重量值 < 10 且状态变为 "off"
   - 包含在床时长统计
   - 置信度计算：基于重量、状态、心率变化

3. **评分体动事件** (`significant_movement_event`)
   - 体动评分 ≥ 3分（1-10分制）
   - 评分维度：位置变化、心率变化、呼吸率变化
   - 强度计算：|Δ头部| + |Δ胸部|，范围0-10

4. **定期测量快照** (`measurement_snapshot`)
   - 每5分钟保存一次完整测量数据
   - 包含：心率、呼吸率、呼吸暂停次数、尿湿状态
   - 仅在上床状态下保存

**数据降噪算法**
- 心率异常值过滤：30-200 BPM范围外设为0
- 呼吸率异常值过滤：5-60次/分钟范围外设为0
- 重量值验证：-1-20范围外设为-1
- 状态值验证：仅接受on/off/mov/call

**传统翻身检测**（作为体动的一部分）
```rust
// 位置变化算法
Value = |头部_后 - 头部_前| + |胸部_后 - 胸部_前|
if Value > threshold {
    检测到翻身事件
}
```

### 4. 数据存储层

**双数据类型存储**
- `mattress_status`: 存储完整的床垫状态数据
- `turn_over_event`: 单独存储翻身事件，便于分析

**数据归属策略**
- 写入时确定数据归属（subject_id）
- 不受后续绑定变化影响
- 支持设备自动注册和患者绑定

## 性能优化

### 1. 内存管理
- 使用固定容量缓冲区（4KB）
- VecDeque维护最近4个位置数据
- 避免频繁内存分配

### 2. 并发处理
- 每个TCP连接独立任务处理
- 异步I/O，非阻塞数据接收
- 连接异常自动清理

### 3. 错误处理
- 数据包级别错误隔离
- 单个数据包错误不影响连接
- 详细的错误日志记录


```bash
APP_TCP_PORT=5858
APP_TCP_ENABLED=true
```

## 使用示例

### 1. 启动服务
```bash
cargo run
```

### 2. 设备连接
智能床垫设备通过TCP连接到服务器端口5858，按照协议格式发送数据。

### 3. 数据查询
通过HTTP API查询床垫数据和翻身事件：
```bash
# 查询最新床垫状态
curl http://localhost:8000/api/v1/data/latest?device_id={device_id}&data_type=mattress_status

# 查询翻身事件
curl http://localhost:8000/api/v1/data/query?device_id={device_id}&data_type=turn_over_event
```
